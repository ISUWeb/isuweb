# RESTful Web Service

In this lesson we will create a program that we can interact with over the internet.

# Our goal

Our goal here is to create a program that we can send data to and receive a response. To do this we will create a RESTful web service.

# RESTful web service?

A web service is a program that can listen and respond to request over the internet, which are sent to a particular endpoint (e.g. 'https://google.com'). A RESTful web service is simply a web service that uses the HTTP methods to denote the type of action we want to perform. For instance, if all we want to do is retrieve data, we would send a GET request to whatever endpoint.

To create our web service we will use Spring Boot, a popular option for Java.

# Spring Boot

Spring Boot is a framework for creating web services. That is, it lays the foundation for everything we need to receive and respond to requests over the internet. All we need is to somehow include it into our project. A dependency manager allows us to do just this.

# Dependency managers

Maven is yet another tool that facilitates our lives as developers. It will fetch any dependencies we need (and any dependencies our dependencies need!) so that we don't need to worry about downloading someone else's code into our project. We simply state what libraries we are dependant upon and maven takes care of the rest. Alternatively, we can use Gradle to accomplish the same thing.

TIP

Maven does much more than this.

# Maven configuration

Eclipse will also create a file called pom.xml. This file contains any configuration for maven and where we will declare our dependencies.

Admitedly there is a lot of things going on here, but add the following contents to your pom.xml file, according to the instructions from Building a RESTful Web Service.

<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.0.5.RELEASE</version>
</parent>

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-test</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>com.jayway.jsonpath</groupId>
        <artifactId>json-path</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>

<properties>
    <java.version>1.8</java.version>
</properties>


<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>
    </plugins>
</build>

<repositories>
    <repository>
        <id>spring-releases</id>

<pluginRepositories>
    <pluginRepository>

# Starting our app

In order for Spring Boot to start our app, create a class App.java with the following content:

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class Application {

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

# Defining endpoints

In Spring, HTTP requests are handled by 'controllers'. These are classes are annotated as @RestController. The name controller comes from the MVC design pattern.

Within a controller we define methods that are executed whenever our web service receives a request to a particular endpoint. We specify the endpoint using the RequestMapping annotation.

@RestController
public class Controller {

    @RequestMapping(value="/hello", method=GET)
    public String hello() {
        return "Hello, World!";
    }
}

# Accepting parameters

Returning a hardcoded message isn't very useful. Let's accept a name as an input and return a custom message.





 
 



@RestController
public class Controller {

    @RequestMapping("/hello")
    public String hello(@RequestParam(value="name") String name) {
        return "Hello, " + name + "!";
    }
}
