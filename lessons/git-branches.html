# Git - Branches

Recall that git lets us save versions of our code that exist in parallel. These are called branches. Whe we created our git repository we got a branch that by default is named master, but we can create others.

TIP

The following command does two things at once. The command checkout is used to switch between branches and the flag b indicates that this is a new branch that needs to be created.

 


~/git-demo> git checkout -b mundo
Switched to a new branch 'mundo'
1
2

Now that we are on a new branch we can make changes independent of what's happening on the master branch. Let's change our Hello.java to print "Hola, Mundo!".

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hola, Mundo!");
    }
}

Create a commit with this change. We will not walk you through it this time. You're on your own.

After this is done, we can change our branch to master. Our change will seemingly dissapear and our code will look exactly as it did before. But don't dispair, our work is not gone. It's still saved on the mundo branch. Git is just showing us the version that is on master.

 




~/git-demo> git checkout master
Switched to branch 'master'
M       Hello.java
~/git-demo>

So suppose that we are not aware of the change being done in the other branch (perhaps it was done by someone else!) and we refactor our program by extracting the hardcoded message to a private variable. Do this and create a commit. We'll wait.

At this point both of our branches have a commit in common, the first commit we created. On top of this commit they each have a diffrent change. How can we apply the work done on mundo onto the work done on master, you ask?

 




~/git-demo> git merge mundo
Auto-merging Hello.java
CONFLICT (content): Merge conflict in Hello.java
Automatic merge failed; fix conflicts and then commit the result.

Unfortunately git was unable to automatically merge the two branches. But we can resolve the conflict manually.

You will notice that Hello.java now looks different.

 










public class Hello {
        private static final String MESSAGE = "Hello, World!";
        public static void main(String[] args) {

                System.out.println(MESSAGE);

                System.out.println("Hola, Mundo!");

This is git noting the conflicting change. It appears that someone change the message, while we extracted the message to a variable. So it seems like we want line 5, but we need to update line 2 with the translated message. After making this change our code now looks like this:

public class Hello {
        private static final String MESSAGE = "Hola, Mundo!";
        public static void main(String[] args) {
                System.out.println(MESSAGE);
        }
}
